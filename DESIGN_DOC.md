# iPhone 设备信息应用 - 设计方案

## 一、用户群体分析

### 1.1 核心用户画像

#### **硬核游戏玩家** (35%)
- **特征**:
  - 玩《原神》《崩坏：星穹铁道》等大型手游
  - 对设备性能极度敏感
  - 关注降频、发热、掉帧问题
  - 经常查看CPU/GPU占用率

- **核心需求**:
  - 实时温度监控（避免过热降频）
  - 发热指数（游戏时温度趋势）
  - 智能降温模式（自动降频但不卡顿）
  - 性能模式切换（极致性能/平衡/省电）
  - 帧率监控（实时FPS显示）

#### **数码科技爱好者** (25%)
- **特征**:
  - 关注最新芯片参数
  - 喜欢对比设备性能
  - 关注屏幕素质、电池健康
  - 经常刷机、越狱

- **核心需求**:
  - 详细的硬件参数（SoC型号、制程工艺）
  - 屏幕信息（供应商、亮度、刷新率）
  - 电池详细信息（循环次数、健康度）
  - 生产日期、保修状态
  - 系统版本、内核信息

#### **开发者/技术人员** (20%)
- **特征**:
  - iOS应用开发者
  - 需要测试不同设备性能
  - 关注设备兼容性

- **核心需求**:
  - 设备型号识别
  - 系统版本和架构
  - 性能基准测试
  - 内存使用情况
  - 存储空间详情

#### **普通用户** (20%)
- **特征**:
  - 不太懂技术参数
  - 想了解设备基本信息
  - 关注二手设备交易

- **核心需求**:
  - 简单易懂的信息展示
  - 电池健康度
  - 存储空间
  - 设备真伪验证
  - 二手设备估值参考

**设计要点**:
- **温度显示**: 实时SoC温度，颜色编码
  - 绿色 (<40°C): 正常
  - 黄色 (40-45°C): 温热
  - 橙色 (45-50°C): 发热
  - 红色 (>50°C): 过热警告

- **发热指数**: 综合评估
  - 计算公式: (温度 × 0.6) + (CPU占用 × 0.3) + (功耗 × 0.1)
  - 显示: 舒适 / 温热 / 发热 / 过热

- **快速操作按钮**:
  - 🎮 游戏模式
  - ❄️ 智能降温
  - 🔋 省电模式
  - ⚡ 极致性能

### 2.2 实时监控区域

```swift
// 温度监控服务
class ThermalService: ObservableObject {
    @Published var currentTemperature: Double = 0
    @Published var thermalState: ThermalState = .nominal
    @Published var发热指数: Double = 0

    enum ThermalState {
        case nominal      // 正常 <40°C
        case fair         // 温热 40-45°C
        case serious      // 发热 45-50°C
        case critical     // 过热 >50°C
    }

    func startMonitoring() {
        // 实时监控温度变化
        Timer.publish(every: 1.0, on: .main, in: .common)
            .autoconnect()
            .sink { _ in
                self.updateTemperature()
                self.update发热指数()
            }
            .store(in: &cancellables)
    }
}
```

### 2.3 智能性能调度

```swift
// 性能模式服务
class PerformanceModeService: ObservableObject {
    enum PerformanceMode {
        case balanced      // 平衡模式 - 默认
        case gaming        // 游戏模式 - 性能优先
        case powerSave     // 省电模式 - 延长续航
        case coolDown      // 降温模式 - 控制发热
        case extreme       // 极致性能 - 不限制
    }

    @Published var currentMode: PerformanceMode = .balanced

    func applyMode(_ mode: PerformanceMode) {
        switch mode {
        case .gaming:
            // 游戏模式: 提升GPU频率，限制CPU降频阈值
            setCPUPriority(high)
            setGPUPriority(maximum)
            setThermalThreshold(48°C) // 48°C才开始降频
            enableProMotion(true)

        case .coolDown:
            // 降温模式: 主动降频，降低发热
            setCPUPriority(low)
            setGPUPriority(medium)
            setThermalThreshold(42°C) // 42°C就开始降频
            limitBrightness(80%)
            enableProMotion(false)

        case .powerSave:
            // 省电模式: 降低功耗
            setCPUPriority(medium)
            setGPUPriority(low)
            disableBackgroundTasks()
            limitBrightness(70%)

        case .extreme:
            // 极致性能: 不限制，可能严重发热
            setCPUPriority(maximum)
            setGPUPriority(maximum)
            setThermalThreshold(55°C) // 几乎不降频
            disableThermalThrottling()

        case .balanced:
            // 平衡模式: 系统默认
            resetToDefault()
        }
    }
}
```

**技术实现注意**:
- iOS不允许应用直接控制CPU/GPU频率
- 可行方案:
  1. 检测和建议模式（显示推荐设置）
  2. 通过降低屏幕亮度间接降温
  3. 提示用户关闭后台应用
  4. 引导用户到系统设置
